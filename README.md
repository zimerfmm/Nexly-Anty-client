## Что это

Nexly Antidetect — настольное приложение для параллельной работы с множеством изолированных браузерных профилей. Используется для арбитража, SMM-менеджмента, парсинга, QA-автоматизации, тестирования и любых задач, где нужны **чистые независимые браузерные сессии** с возможностью автоматизации.

Внутри — модифицированный на уровне исходного кода Chromium, визуальный редактор скриптов без программирования, встроенное решение капч и инструменты отладки.

## 📥 Установка

### Скачать установщик

**→ [Последняя версия на странице релизов](https://github.com/zimerfmm/Nexly-Anty-client/releases/latest)**

На странице релиза скачай файл `NexlyAntidetectSetup.exe`.

### Запустить

1. Дважды кликни по скачанному `.exe`.
2. Если Windows SmartScreen предупредит — нажми **«Подробнее»** → **«Выполнить в любом случае»**. Приложение новое, Microsoft ещё не успел накопить репутацию подписи.
3. Пройди через мастер установки (путь по умолчанию — `C:\Program Files\Nexly Antidetect\`).
4. После установки на рабочем столе появится ярлык.

### Первый запуск

- Запусти **Nexly Antidetect** с ярлыка.
- Войди в аккаунт (регистрация — на сайте [nexly-anty.com](https://nexly-anty.com/)).
- Готово — можно создавать профили.

Приложение **автоматически проверяет обновления** при каждом запуске. Когда выйдет новая версия, ты получишь предложение обновиться в один клик.

## 🎯 Возможности

### 🛡 Антидетект-ядро

Антифингерпринт работает **на уровне исходного кода Chromium**, а не через JS-инъекции поверх страницы. Это закрывает целый класс проверок, которые умеют обнаруживать обычные антидетекты через `iframe.contentWindow`, cross-origin проверки, доступ к нативным свойствам.

**Результат на pixelscan.net: 0% противоречий** в fingerprint-аудите.

Настраивается: Canvas, WebGL, AudioContext, шрифты, WebRTC, User-Agent, screen, platform, hardware concurrency, device memory, timezone, locale. Headless полностью скрыт — `navigator.webdriver`, `window.chrome`, headers, маркеры — всё корректно.

### ⚙️ Параллельные процессы

Каждый профиль — это связка **Chrome + Node.js**, работающая как единый процесс. Приложение запускает их параллельно, контролирует память, перезапускает при падении, собирает логи в общую консоль.

- Запуск N профилей **одновременно** (лимит по тарифу).
- **Раунды** — несколько проходов скрипта по списку профилей.
- Cookies и сессии сохраняются **перед** закрытием браузера.
- Мониторинг памяти, Chrome watchdog, graceful shutdown.

### 🧩 Визуальная автоматизация без программирования

Встроенный **блочный редактор скриптов** — собираешь сценарий из готовых блоков мышкой, приложение генерирует Puppeteer-код автоматически.

- **15 категорий** блоков: навигация, клики, ввод, ожидания, проверки, логика, капча, файлы, БД, HTTP-запросы, браузер, текст, числа, коллекции, переменные.
- Два режима — **визуальный** (drag-and-drop) и **код** (читаемый Puppeteer-JS).
- Переменные скрипта с типами.
- **Кастомные функции** — создай свой блок из других блоков.

### 🤖 Решение капч из коробки

- **reCAPTCHA v2** (чекбокс и invisible)
- **reCAPTCHA v3** (score-based)
- **Cloudflare Turnstile**

Работает через 2captcha API — просто вставляешь ключ в настройки.

### 🐞 Отладка скриптов

Уникальная возможность — **запуск скрипта в режиме тестирования** с полной видимостью происходящего:

- **Live-скриншоты** из headless Chrome, обновляются на каждом шаге.
- **Breakpoints** — останавливаешься на любом блоке, смотришь значения переменных.
- **Пошаговое выполнение** — один блок за раз.
- **Визуализация графа** — текущий выполняющийся блок подсвечивается.
- Console-вывод скрипта рядом.

### 🎯 Конвертер HTML → CSS/XPath

Вставляешь фрагмент HTML → получаешь **готовые селекторы** с оценкой устойчивости (1-5 звёзд).

Конвертер умный: понимает `id`, `data-testid`, `aria-label`, структуру родитель-потомок, sibling-оси, текстовое содержимое. **Отсеивает динамические классы** (styled-components, css-modules, JSX, Svelte, Emotion, Tailwind JIT, хеши) — они нестабильны, и селектор на них ломается при следующем деплое сайта.

### 🌐 Прокси

- **Параллельная проверка** работоспособности прокси.
- Поддержка HTTP, HTTPS, SOCKS5 + авторизация.
- **Массовые операции**: назначить прокси на пустые профили, равномерное распределение, удалить все.

### 🛠 Страница администрирования

Отдельный **рабочий стол под бота** — кастомные кнопки-биндинги, свои метрики, отдельная админская консоль.

### ☁️ Облако и бэкапы

- Cookies каждого профиля **автоматически заливаются в облако** перед закрытием браузера.
- **Восстановление** профилей из облака — при переустановке или на новом компьютере.
- **Импорт/экспорт** профилей в файл (light и heavy форматы).

## 🧩 Системные требования

| | Минимум | Рекомендуется |

| **ОС** | Windows 10 (x64) | Windows 11 |

## 🆙 Обновления

Приложение проверяет обновления при каждом запуске. Когда доступна новая версия, появляется диалог:

> **Доступно обновление: X.Y.Z → X.Y.Z+1. Обновить?**

Нажимаешь **«Обновить»** — новый installer скачивается (с проверкой SHA-256 на целостность), запускается, приложение закрывается, установка идёт автоматически.

Можно нажать **«Позже»** — обновление отложится до следующего запуска.

## ❓ Частые вопросы

**Приложение безопасное?**
Установщик и клиент не содержат вредоносного кода. Проверка целостности встроена — каждое обновление верифицируется по SHA-256 перед запуском.

**Почему Windows SmartScreen предупреждает?**
Это стандартное поведение для нового установщика, который Microsoft ещё не видел в массе. Проходит со временем, когда накапливается статистика установок. Пока — «Подробнее» → «Выполнить в любом случае».

**Работает ли на Windows 7/8?**
Нет. Требуется Windows 10 (1809+) или Windows 11.

**Где хранятся профили?**
В `%APPDATA%\NexlyAntidetect\` или другом каталоге, указанном в настройках. Каждый профиль — отдельная папка со своим Chrome user data, cookies и конфигом.

**Можно использовать одну лицензию на нескольких компьютерах?**
Зависит от тарифа. Подробности — на сайте [nexly-anty.com](https://nexly-anty.com/).

**Есть пробный период?**
Да. Условия — на сайте.
